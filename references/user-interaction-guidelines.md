# 用户交互规范

> **适用范围**：所有 devflow-kit stage 的用户确认环节
> **参考来源**：`references/prompts/0-confirm.md`

## 核心原则

1. **选项优先**：提供选项按钮，减少文字输入
2. **逐项确认**：多个问题每轮最多 3 个，等用户回答再继续
3. **默认明确**：标明推荐选项
4. **单点聚焦**：一轮只问一类问题，避免信息超载

---

## 一、单问题确认格式

### ✅ 正确示例

```markdown
---
**请选择下一步：**

[1] ✅ 确认，继续下一阶段 ← 推荐
[2] 查看产物详情
[3] 需要修改

请回复 1/2/3。
```

### ❌ 错误示例

```markdown
请确认产物内容是否正确：
- ✅ 确认，继续下一阶段
- ❌ 需要修改：`<具体修改内容>`
```

**问题**：用户需要输入完整文字，增加认知负担。

---

## 二、多问题确认格式（参考 0-confirm 步骤 5）

### 原则

> **每轮最多 3 个问题，等用户回答再继续**

### Step 1: 列出问题（不超过 3 个）

```markdown
📍 当前阶段：<stage-name> · <阶段描述>
🎯 本步目标：<本次确认的目标>

我需要确认以下问题：

---
**问题 1**：<第一个问题>
[1] <选项A> ← 推荐
[2] <选项B>
[3] 其他（请说明）

**问题 2**：<第二个问题>
[1] <选项A> ← 推荐
[2] <选项B>

**问题 3**：<第三个问题>
[1] <选项A> ← 推荐
[2] <选项B>

请逐个回复，如 "1, 2, 1" 或 "问题1选1，问题2选2..."
```

### Step 2: 用户回答后确认

```markdown
✅ 收到你的选择：
- 问题 1：<选项A>
- 问题 2：<选项B>
- 问题 3：<选项A>

---
**确认以上选择：**

[1] ✅ 确认无误，继续 ← 推荐
[2] 需要修改

请回复 1/2。
```

## 三、阶段完成确认格式（参考 0-confirm 阶段完成声明）

### 产物确认

```markdown
✅ <阶段名称> 完成
📝 产物：.devflow-kit/<req-id>/<产物文件>
📊 STATE.md 阶段进度已更新：[x] <阶段> → <产物>

请确认产物内容：
- <关键点1>: <值>
- <关键点2>: <值>
- <关键点3>: <值>

---
**请选择下一步：**

[1] ✅ 确认，继续下一阶段 ← 推荐
[2] 查看产物详情
[3] 需要修改

请回复 1/2/3。
```

### 发布确认

```markdown
---
## 🚀 准备发布

### 发布内容
- 版本：vX.X.X
- 新增功能：...

### 发布检查
- [x] 测试通过
- [x] Code Review 通过
- [x] 文档已更新

---
**请选择：**

[1] ✅ 确认发布 ← 推荐
[2] 推迟发布

请回复 1/2。
```

## 四、架构级变更检测（参考 0-confirm 步骤 2）

### 命中后停下来反问

```markdown
🏛️ 检测到本次描述涉及**项目级架构变更**：
   - 命中信号：<具体信号>

这种规模建议先跑架构梳理：

---
**请选择：**

[1] ✅ 先跑架构梳理（推荐）
[2] ⚠️ 我清楚影响，继续当前流程
[3] ↩️ 这其实是需求级局部改造（请说明理由）

请回复 1/2/3。无确认前不进后续步骤。
```

## 五、视觉调性选择（参考 0-confirm 步骤 4）

### 红线：本步不出其他问题

```markdown
📍 项目：<项目名>
🎯 一句话：<业务描述>

### 视觉调性选项

[1] <调性1> - <关键词> ← 推荐（理由）
[2] <调性2> - <关键词>
...
[9] <调性9> - <关键词>

❌ 不推荐：<调性X>（理由）、<调性Y>（理由）

请回复数字（如 "6"）或描述你想要的感觉，选定后我再问其他问题。
```

## 六、特殊情况处理

### 用户选择"需要修改"

```markdown
📝 请说明需要修改的内容：

直接输入修改意见，例如：
- "任务3的时间预估需要调整"
- "设计文档缺少数据库设计"
```

用户输入后：
```markdown
✅ 已记录修改意见：<用户输入>

正在重新生成...
```

### 用户选择"查看产物详情"

```markdown
---
**产物详情：**

### <产物名称>

<产物内容摘要或全文>

---
**请选择下一步：**

[1] ✅ 确认，继续 ← 推荐
[2] 需要修改

请回复 1/2。
```

## 七、选项编号规则

| 编号 | 含义 | 标记 |
|---|------|---|
| 1 | 推荐选项 | `✅ ... ← 推荐` |
| 2 | 备选选项 | `⚠️ ...` |
| 3 | 查看详情 | `查看产物详情` |
| 4 | 需要修改 | `需要修改` |
| 5+ | 其他选项 | 根据具体情况添加 |

**标记规则**：
- 推荐选项：`✅ ... ← 推荐`
- 警告选项：`⚠️ ...`
- 用户可直接输入数字或选项文字
