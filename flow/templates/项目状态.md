**入场检查清单**：
```

□ brownfield 项目：扫描已完成 或 用户已明确跳过

□ 此节已填写完整
```

## 🔄 压缩恢复指引

> ⚠️ **会话压缩后，AI 必须先读本节恢复上下文，不能假装从未中断。**

### 恢复步骤

```
Step 1 ─ 读本文件
         └─ 获取：活跃 Req、当前阶段、当前 Task、模式、产物清单、入场扫描状态

Step 2 ─ 检查入场扫描
         └─ 若「扫描状态」≠「已完成」且项目为 brownfield → 先完成扫描

Step 3 ─ 读当前阶段产物
         └─ 查下方「阶段进度」找到已完成最后阶段
         └─ 读该阶段产物文件，恢复上下文

Step 4 ─ 输出恢复摘要（必须）
         ┌─────────────────────────────────────────────────────────────┐
         │  🔄 会话恢复摘要                                            │
         ├─────────────────────────────────────────────────────────────┤
         │  恢复自：<产物文件名>                                       │
         │  当前阶段：<阶段名>                                         │
         │  当前任务：<Task ID 或 "—">                                 │
         │  模式：<Fast/Standard/Strict>                               │
         │  已完成阶段：<列出已打钩的阶段>                              │
         │  下一步：<具体下一步动作>                                    │
         └─────────────────────────────────────────────────────────────┘

Step 5 ─ 继续执行
         └─ 按「下一步」继续，不跳过任何阶段
```

### 各阶段必加载文件

| 阶段 | 必加载文件 |
|------|-----------|
| 需求确认 | `项目状态.md` |
| ANALYSIS | `00-需求确认.md` |
| DESIGN | `00-需求确认.md` + `01-需求分析.md` + `上下文.md` |
| TASK | `02-方案设计.md` + [`02a-UI设计.md`] |
| DEV | `03-任务拆分.md`（当前 task 块）+ `02-方案设计.md` §0 |
| TEST | `03-任务拆分.md` + `04-开发记录.md` + 各 `*-开发记录.md` |
| REVIEW | `05-测试报告.md` + `git diff` |
| INTEGRATION | 全部产物 + `06-代码审查.md` |

### 恢复后禁止行为

```
❌ 假装从未中断，不输出恢复摘要
❌ 跳过已完成阶段重新做
❌ 忽略产物清单，重新生成已存在产物
❌ 改变已确定模式（除非用户明确要求）
❌ 忘记当前任务中断点，从头开始
❌ 跳过入场扫描检查（brownfield 项目必须确认）
```
