### 入场扫描检查清单

```
进入流程前必须确认：
- [ ] 已判断项目类型（greenfield / brownfield）
- [ ] brownfield 项目：入场扫描已完成 或 用户已明确跳过
- [ ] 上下文文档已存在 或 用户已选择"盲飞"模式
- [ ] 项目状态.md 此节已填写完整
```

### 未完成入场扫描时的处理

```
若「入场扫描状态」≠「已完成」且项目为 brownfield：
  → 停止进入 0-confirm
  → 提示用户：需要先完成入场扫描或明确跳过
  → 用户选择：
     1. 现在执行入场扫描（推荐）
     2. 跳过扫描，AI "盲飞"（不推荐）
     3. 指定现有文档作为上下文（如 AGENTS.md）
```

## 🔄 压缩恢复指引（会话压缩后 AI 必读）

> ⚠️ **当会话被压缩恢复后，AI 必须先读本节，按步骤恢复上下文，不能假装从未中断。**

### 恢复步骤（必须按顺序执行）

```
Step 1: 读本文件（项目状态.md）
   └─ 获取：活跃 Req、当前阶段、当前 Task、模式、产物清单、入场扫描状态

Step 2: 检查入场扫描状态
   └─ 若「入场扫描状态」≠「已完成」且项目为 brownfield → 先完成入场扫描

Step 3: 读当前阶段的产物文件
   └─ 按下方「当前模式 → 阶段进度」找到已完成的最后一个阶段
   └─ 读该阶段的产物文件，恢复上下文

Step 4: 输出恢复摘要（必须）
   ┌─────────────────────────────────────────────────────────────┐
   │  🔄 会话恢复摘要                                            │
   ├─────────────────────────────────────────────────────────────┤
   │  恢复自：<产物文件名>                                       │
   │  当前阶段：<阶段名>                                         │
   │  当前任务：<Task ID 或 "—" >                                │
   │  模式：<Fast/Standard/Strict>                               │
   │  已完成阶段：<列出已打钩的阶段>                              │
   │  下一步：<具体下一步动作>                                    │
   └─────────────────────────────────────────────────────────────┘

Step 5: 继续执行
   └─ 按「下一步」继续，不跳过任何阶段
```

### 恢复时必须加载的文件（按阶段）

| 当前阶段 | 必须加载的文件 |
|----------|----------------|
| 需求确认 | `项目状态.md` |
| ANALYSIS | `00-需求确认.md` |
| DESIGN | `00-需求确认.md` + `01-需求分析.md` + `上下文.md` |

### 恢复后禁止的行为

- ❌ 假装从未中断，不输出恢复摘要
- ❌ 跳过已完成的阶段重新做
- ❌ 忽略产物清单，重新生成已存在的产物
- ❌ 改变已确定的模式（除非用户明确要求）
- ❌ 忘记当前任务的中断点，从头开始
- ❌ 跳过入场扫描检查（brownfield 项目必须确认）
