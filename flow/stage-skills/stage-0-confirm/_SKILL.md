## 执行流程

### Step 1: 扫描可用前置产物

**⚠️ 强制规则**：必须先扫描所有可能的前置产物，根据实际存在情况决定读取策略。

#### 1.1 扫描主流程产物

| 产物文件 | 存在性 | 优先级 | 提取内容 |
|---------|--------|--------|----------|
| 项目状态.md | ✅/❌ | 🔴 必须 | 活跃req、中断任务、当前阶段 |
| 上下文.md | ✅/❌ | 🟡 建议 | 技术栈、编码规范、禁动清单 |
| 经验总结.md | ✅/❌ | 🟢 可选 | 历史教训、最佳实践 |

**扫描结果输出**：
```markdown
✅ 检测到 项目状态.md → 提取：无活跃req
✅ 检测到 上下文.md → 提取技术栈：Vue3 + TypeScript
❌ 未检测到 经验总结.md → 使用通用最佳实践
```

#### 1.2 分级读取策略

**🔴 必须读取**（缺失会阻塞流程）：
- **项目状态.md**：提供当前进度和中断任务
  - 如果缺失 → 创建新项目状态文件

**🟡 建议读取**（缺失采用降级策略）：
- **上下文.md**：提供项目技术背景
  - 如果缺失 → 使用通用技术假设，在产物中增加"技术假设"章节

**🟢 可选读取**（补充信息）：
- **经验总结.md**：提供历史教训
  - 如果缺失 → 使用通用需求澄清问题

#### 1.3 降级策略

**如果某个产物不存在**：

1. **上下文.md 缺失**：
   ```
   ⚠️ 警告：缺少上下文.md，无法获取项目技术背景
   → 降级方案：在00-需求确认中增加"技术假设"章节
   → 询问用户："是否需要先运行 I-intel-scan 生成上下文？"
   ```

2. **经验总结.md 缺失**：
   ```
   ℹ️ 提示：缺少经验总结.md，使用通用需求澄清问题
   → 提醒：建议在完成后将本次经验追加到经验总结.md
   ```

#### 1.4 信息提取摘要

**输出格式**：
```markdown
### Step 1 输出：前置信息摘要

**已提取的关键信息**：

1. **项目层面**（来自项目状态）：
   - 活跃req：<无/REQ-XXX>
   - 中断任务：<无/TXX>

2. **技术层面**（来自上下文，如存在）：
   - 技术栈：<Vue3 + TypeScript>
   - 编码规范：<PascalCase组件名>

**下一步**：基于上述信息进行需求澄清
```

### Step 2: 需求澄清

使用 `idea-refine` skill 进行需求反问：
- 目标用户是谁？
- 核心功能是什么？
- 成功标准是什么？
- 有哪些约束条件？

### Step 3: 模式判定

根据需求复杂度判定执行模式：

| 维度 | Fast | Standard | Strict |
|------|------|----------|--------|
| 改动规模 | ≤2文件，<50行 | 中等功能 | 大型重构/高风险 |
| 风险等级 | 低 | 中 | 高 |
| 团队熟悉度 | 非常熟悉 | 一般 | 不熟悉/新技术 |

**输出格式**（必须按此格式）：
```markdown
🎯 建议模式：<Standard / Strict>
   理由：<简要说明为什么是这个模式>

   📋 <模式> 完整流程：
   <流程步骤简述>
   
   产物：<关键产物列表>

   其他可选模式：
   • Fast — ≤2 文件、<50 行、低风险
   • Strict — 高风险/生产敏感/架构或数据影响

   请确认或选择其他模式：
   1. ✅ <建议模式>（推荐）
   2. Fast
   3. Strict
```

**注意**：
- 必须包含"理由"字段，说明推荐该模式的原因
- 必须展示建议模式的完整流程和产物清单
- 其他模式只需一句话简介
- 用户可选择覆盖AI推荐

### Step 4: 完整性自检

**检查是否充分利用了前置产物**：

- [ ] 是否读取了项目状态.md？
- [ ] 是否参考了上下文.md中的技术背景（如存在）？
- [ ] 对于缺失的产物，是否采用了合理的降级策略？
- [ ] 需求澄清问题是否全面（目标用户/核心功能/成功标准/约束条件）？
- [ ] 模式判定是否基于充分信息？

**如果发现遗漏**：
→ 回到Step 1重新读取
→ 或在本阶段产物中注明"因缺少XX产物，采用YY假设"

### Step 5: 读取模板并提取段落清单

**⚠️ 强制规则**：必须使用 `read_file` 工具读取模板文件。

```python
# 伪代码示例
read_file("flow/templates/00-需求确认.md")
```

**从模板中提取必填段落清单**（模板 L16-30）：
```
□ Why（为什么做）
□ What（做什么）
□ 一句话总结
□ 背景与动机
□ 预期成果
□ 视觉调性（前端项目必填）
□ 影响面
□ 影响项表格
□ 范围排除（至少1条）
□ 验收线
□ 风险与未知
□ 架构层影响声明（如有）
```

**注意**：
- "架构层影响声明"如无内容，应保留段落标题并填写"无架构层影响"
- 不得因为"无内容"而删除该段落
