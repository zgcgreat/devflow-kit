# 团队偏好

本文件用于记录持久的协作偏好、沟通边界和团队特定的工作约定。

## 条目模板

```md
### 偏好：<简短标题>
- id: preference-YYYY-MM-DD-<标识>
- type: team_preference
- status: active
- confidence: verified
- last_updated: YYYY-MM-DD
- source:
- owner:
- review_after:

偏好内容：

存在原因：

如何应用：
```

## 说明

- 本文件用于持久偏好，而非会话特定的请求。
- 如果偏好发生变化，更新现有条目或标记为 `superseded`。
- 持久化条目应始终包含 `id`、`status`、`confidence`、`source`、`last_updated` 和 `review_after`。
