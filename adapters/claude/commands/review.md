# /review — 四轮代码审查

## 快速模式

用户可指定审查范围：
- `/review` — 完整四轮审查（默认）
- `/review quick` — 快速模式（仅第一轮+第二轮）
- `/review security` — 安全专项（仅第二轮安全检查）
- `/review ui` — UI 专项（仅第三轮）

## Critical 问题批量处理

发现多个 Critical 时，**一次性列出所有问题**，然后统一询问：

```
🔴 发现 3 个 Critical 问题：

1. [R1-认知过载] src/utils.ts:42 — 函数超过 80 行
2. [AC-02] 缺少测试覆盖
3. [Security] SQL 注入风险

请选择处理方式：
- 输入 "fix 1,2" → 仅修复问题 1 和 2
- 输入 "fix all" → 修复全部
- 输入 "accept 3" → 接受问题 3 的风险（需确认）
- 输入 "discuss 1" → 讨论问题 1 的详情
```

## 输出

- `.specs/<req-id>/06-代码审查.md`
- 修复任务追加到 `03-任务拆分.md`（如有 Critical/Major）
