# 02-design — 方案设计

> **Req ID**: user-login  
> **创建时间**: 2026-05-16 15:55:00

## 0. 技术栈选定

- **认证库**: jsonwebtoken (JWT), bcryptjs (密码加密)
- **验证库**: joi (请求参数验证)
- **中间件**: express-rate-limit (速率限制)

## 0.5 既有架构对齐

### 触碰模块
- 新增: src/auth/ (认证模块)
- 新增: src/models/User.js (用户模型)
- 修改: src/app.js (注册路由)

### 禁动清单
- ❌ 不修改 src/config/database.js
- ❌ 不修改现有用户相关接口

## 1. 架构设计

### 1.1 认证流程
```
注册: 用户输入 → 验证 → 密码加密 → 保存数据库 → 返回成功
登录: 用户输入 → 验证 → 查询用户 → 密码比对 → 生成JWT → 返回Token
刷新: Refresh Token → 验证签名 → 生成新Access Token → 返回
```

## 2. 数据模型

### User 模型
```javascript
{
  email: String (unique, required),
  password: String (hashed, required),
  refreshToken: String (optional),
  rememberMeExpiry: Date (optional),
  createdAt: Date,
  updatedAt: Date
}
```

## 9. 设计决策沉淀

### 决策 1: JWT 存储位置
**决策**: Access Token 存储在内存，Refresh Token 存储在 HttpOnly Cookie  
**理由**: 防止 XSS 攻击，Cookie 设置 HttpOnly 和 Secure 标志  
**影响**: 前端需要使用 withCredentials 发送 Cookie
```
