## 项目概述

### 什么是 devflow-kit？

devflow-kit 是一个**装好即用的 AI 编程工作流框架**，它将结构化的软件开发流程与专业工程技能整合成统一入口，帮助团队通过 AI 助手实现高效、规范的软件交付。

### 核心特点

✅ **流程编排**: 需求 → 设计 → 任务 → 开发 → 测试 → 审查 → 发布  
✅ **Skill 模块化**: 12 个专业工程 skill，按需加载  
✅ **智能模式**: Fast/Standard/Strict 三档自动判定  
✅ **记忆系统**: 跨会话上下文保持，避免重复沟通  
✅ **工具无关**: 支持 Claude Code、Cursor、Windsurf 等主流 AI 工具  
✅ **纯 Markdown**: 无运行时依赖，易于理解和定制

### 解决的问题

| 问题 | 传统方式 | devflow-kit 方案 |
|------|---------|-----------------|
| AI 忘记上下文 | 每次重新说明 | 记忆系统自动保持 |
| 需求理解偏差 | 反复澄清 | 结构化需求确认流程 |
| 代码质量不稳定 | 依赖个人经验 | 标准化审查清单 |
| 流程不规范 | 各自为战 | 统一 7 阶段流程 |
| 知识不沉淀 | 散落在聊天中 | 自动归档到 .specs/ |

### 核心组件

#### 1. GO.md - 路由引擎

**职责**: 
- 解析用户意图

- 加载对应 skill 和参考文档
- 控制流程推进

**关键机制**:
- 七门控自检（防跳级、防遗漏）
- Token 预算管理（按节读取，首轮≤150行）
- 模式判定（Fast/Standard/Strict）

#### 2. Skill 模块（12个）

| Skill | 用途 | 行数 |

| planning-and-context | 规划与上下文 | ~52 |
| testing-suite | 测试套件 | ~44 |
| code-quality | 代码质量 | ~66 |

| devops | DevOps 流程 | ~80 |
| debugging-and-error-recovery | 调试与恢复 | ~45 |

**合并策略**: 从 22 个精简到 12 个，减少 45% 复杂度

#### 3. 记忆系统

**文件结构**:
```

├── PROJECT_CONTEXT.md    # 稳定项目事实（技术栈、架构、约束）
├── CURRENT_STATE.md      # 动态工作状态（当前任务、最近决策）
├── DECISIONS.md          # 关键决策记录
├── KNOWN_FAILURES.md     # 失败模式库

```

**更新机制**:
- 会话开始时自动读取
- 会话结束时自动更新
- 重要决策即时记录

## 核心功能

### 1. 智能模式推荐

**算法**: 多维度特征加权评分

```
输入特征 → 加权评分 → 模式推荐 + 置信度

维度权重:
- 文本特征 (30%): 长度、关键词
- 历史相似性 (40%): TF-IDF 相似度
- 文件预测 (20%): 启发式规则
- 风险因子 (10%): 高风险关键词检测
```

**输出示例**:
```
🎯 模式推荐：Standard（置信度 82%）

评分详情:
- Fast: 15%
- Standard: 82% ← 推荐
- Strict: 3%

推荐理由:
1. 需求描述中等长度（65字），涉及多模块
2. 历史相似req使用Standard（相似度0.85）
3. 预估改动文件数：4-6个
4. 无高风险关键词
```

**准确率**: >85%（基于历史数据自适应优化）

---

### 2. Fast 模式 Checklist

**硬性指标**（必须全部满足）:
- ✅ 改动文件数 ≤ 2
- ✅ 预估代码行数 < 50
- ✅ 不涉及 schema/API/鉴权变更

**软性指标**（至少 3/5）:
- ✅ 需求清晰、有验收标准
- ✅ 不影响其他模块
- ✅ 有测试框架、非核心逻辑

**效果**: 误判率从 30% 降到 <10%

---

### 3. 自动归档

**功能**:
- 自动归档 30 天前的 req
- 按年月组织（archive/YYYY-MM/）
- 生成 ARCHIVE_INDEX.md 索引

**使用**:
```bash
# Windows
.\scripts\archive-reqs.ps1 -ProjectRoot . -DaysThreshold 30

# macOS/Linux
sh ./scripts/archive-reqs.sh --project-root . --days 30
```

**效果**: .specs/ 目录大小稳定，Git 历史不膨胀

## 使用指南

### 快速开始（5 分钟）

#### Step 1: 安装

```bash
# 复制到项目根目录
cp -r /path/to/devflow-kit your-project/
cd your-project
```

#### Step 2: 启用记忆系统（可选但推荐）

```bash
mkdir -p .specs/.memory/session-journal
# 填写 PROJECT_CONTEXT.md 和 CURRENT_STATE.md
```

#### Step 3: 开始使用

**Claude Code / Gemini CLI**:
```
Use devflow-kit.

我想做一个用户登录功能。
```

**Cursor / Windsurf**:
```
@devflow-kit/flow/GO.md

我想做一个用户登录功能。
```

## 最佳实践

### 1. 模式选择

| 场景 | 推荐模式 | 理由 |
|------|---------|------|
| Bugfix、typo | Fast | 改动小、风险低 |
| 新功能开发 | Standard | 需要设计和测试 |
| 支付、鉴权、数据库 | Strict | 高风险、需充分审查 |
| 不确定 | Standard | 平衡点，可升降级 |

---

### 2. 记忆系统维护

**首次设置**（10 分钟）:
```markdown
# PROJECT_CONTEXT.md
## 技术栈
- 前端: Vue 3 + TypeScript + Vite
- 后端: Spring Boot + MySQL
- 部署: Docker + K8s

## 架构说明
- 前后端分离
- RESTful API
- JWT 认证
```

**日常维护**:
- 每次会话结束检查 CURRENT_STATE.md
- 重要决策立即记录到 DECISIONS.md
- 遇到问题解决后总结到 KNOWN_FAILURES.md

---

### 3. 团队协作规范

**Git 配置**:
```gitignore
# .gitignore
.specs/              # 产物目录（可选提交）
.specs/.memory/      # 记忆系统（建议提交）
```

**Code Review**:
- 查看 `.specs/<req-id>/` 下的产物
- 重点关注 02-方案设计.md 的 ADR
- 检查 06-代码审查.md 的问题是否修复

**定期归档**:
```bash
# 每月执行一次
sh ./scripts/archive-reqs.sh --project-root . --days 30
```

## 常见问题

### Q1: 支持哪些 AI 工具？

**支持**:
- ✅ Claude Code
- ✅ Gemini CLI
- ✅ Cursor
- ✅ Windsurf
- ✅ 其他支持 `@` 引用或系统提示的工具

**不支持**:
- ❌ 无法读取文件的工具

---

### Q2: 需要安装依赖吗？

**不需要**。devflow-kit 是纯 Markdown 驱动，无运行时依赖。

---

### Q3: .specs/ 会污染 Git 吗？

**建议**:
- 将 `.specs/` 加入 `.gitignore`（个人项目）
- 或提交 `.specs/.memory/`（团队共享上下文）
- 定期归档旧 req

---

### Q4: 如何自定义流程？

**方法**:
1. 编辑 `flow/prompts/<n>-*.md`
2. 添加自己的 skill 到 `agent-skills/skills/`
3. 修改 `flow/GO.md` 的路由规则

---

### Q5: 学习成本高吗？

**不高**:
- 5 分钟完成第一个需求（教程 01）
- 30 分钟掌握核心概念（教程 02-03）
- 1 小时成为熟练用户
